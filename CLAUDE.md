# CLAUDE.md

Project context for Claude Code (and other agents) working on this repo.

## What this is

A small public Observable Framework site that visualizes Kalshi market data:
leaderboards, fees, volume, calibration, competitor comparisons, parlay /
forecast stats, etc. The site is deployed to GitHub Pages on every push to
`main`.

Live: https://doboyle95.github.io/kalshi-dashboard/

## Stack

- [Observable Framework](https://observablehq.com/framework/) `^1.4.0`
- Pages live in `src/*.md` and are authored in Markdown with embedded
  ```js fenced code blocks.
- Data lives in `src/data/*.csv`. These CSVs are generated by a separate,
  **private R pipeline** owned by the repo owner — do not try to regenerate
  them from this repo. Treat the CSVs as inputs.
- No backend. Everything renders client-side from the CSVs.

## Working with categories.md

`src/categories.md` contains the Individual Market Leaderboard. It has to turn
raw Kalshi `market_key` / `winner_ticker` / `top_outcome` strings into
human-readable Market / Winner / Highest-vol. strike cells.

The conversion logic lives in these functions in the big `js` block that
starts with `// ── Category colors ──`:

| Function / dict | Purpose |
| --- | --- |
| `CAT_COLORS` | Display-category colors. Sports is split into Football / Basketball / Other sport. |
| `NFL_TEAMS`, `NBA_TEAMS`, `MLB_TEAMS`, `NHL_TEAMS`, `CFB_TEAMS`, `CBB_TEAMS`, `SOCCER_TEAMS`, `CRICKET_TEAMS`, `IPL_TEAMS`, `TENNIS_PLAYERS`, `GOLF_PLAYERS` | Sport-specific code → name lookups. |
| `getTeamsForMarket(mk)` | Routes a `market_key` to the correct team/player dict. Uses prefix matching. **Always use this instead of `ALL_TEAMS`** when the market is identifiable, to avoid cross-sport collisions (e.g. MLB `SEA` → "Mariners", not NFL `SEA` → "Seahawks"). |
| `parseGame(code, teamMap)` | Greedy 2-team parse (longest-match first). |
| `parseTicker(mk)` | Ticker → readable market name for the cases where Kalshi's `market_name` is blank or just echoes the key. |
| `MKT_NAME_FORCE` | **Takes priority** over Kalshi's `market_name`. Use for markets where Kalshi's label is wrong or missing useful context (e.g. World Series games, Super Bowl LX, 2025 Masters). |
| `MKT_NAME_OVERRIDES` | Fallback override used only when both Kalshi `market_name` and `i.market_name` are blank or equal to the ticker key. |
| `WINNER_OVERRIDES` | Hard-coded winner display (keyed on `winner_ticker`). |
| `WINNER_BY_MARKET` | Winner for markets where `winner_ticker` is empty but the market did settle. Keyed on `market_key`. |
| `TOP_OUTCOME_NAMES` | Human names for specific `top_outcome` strings (full ticker, including market_key prefix). |
| `fmtWinner(d)` | Runs `WINNER_OVERRIDES` → strips long rule text → uses `winner` field → strips `::` prefixes → falls back to decoding the suffix of `winner_ticker` via `getTeamsForMarket`. Finally falls back to `WINNER_BY_MARKET`. |
| `fmtStrike(top, mk)` | Runs `TOP_OUTCOME_NAMES` → Fed H0/Hn/Cn → days (`42D` → `42 days`) → date (`26MAR01` → `by Mar 1`) → spread (`SEA4` → `Seahawks -4`) → percentage (**only** for `POPVOTEMOV*` markets) → sport-aware team/player. |
| `getSportDisplayCategory(d)` | Splits Kalshi's `Sports` category into `Football` / `Basketball` / `Other sport` for coloring and filtering. |

### Important correctness rules — do NOT regress these

1. **Sport isolation.** MLB/NBA/NHL/NFL team codes overlap (SEA, LA, TOR, etc.).
   Always look up team codes via `getTeamsForMarket(mk)` — not `ALL_TEAMS` —
   unless the market is genuinely ambiguous.
2. **Percentages.** Only `POPVOTEMOV*` markets use bare numbers that mean
   percent. Bitcoin price strikes and Electoral College margin strikes are
   bare numbers — they are NOT percentages. Do not slap `%` on them.
3. **Fed decisions.** Kalshi labels "Hike 0bps" for holds — we render `Hold`
   via a metadata-fix branch in `fmtWinner`.
4. **Market rule text.** When Kalshi's `winner` column is >50 characters it's
   almost always market-rule text like "If X wins the match…". Strip it (see
   the `rawW.length > 50 ? ""` line).
5. **`market_name` equals `market_key`.** Some Kalshi markets have
   `market_name` populated but it's just an echo of the ticker. Treat those as
   blank — that's what `bestName` does.
6. **Parlays.** Any market_key starting with `KXMVE` is a Kalshi parlay
   ("multi-game extended"). Display as `Parlay`, not the raw ticker.

### When adding a new market / fixing a new ticker

See `TICKER-RULES.md` for the full conversion reference. High-level flow:

1. If Kalshi's `market_name` is correct as-is → do nothing.
2. If Kalshi's `market_name` is wrong/misleading → add to `MKT_NAME_FORCE`.
3. If blank / echoes ticker → add to `MKT_NAME_OVERRIDES`, OR teach
   `parseTicker` a new regex branch if it's a whole family of tickers.
4. If a new sport appears, add a new team/player dict and add a routing
   branch to `getTeamsForMarket`.
5. If the winner cell renders a code like `SEA` instead of `Seahawks`, first
   check the sport routing. If it's a one-off (PGA player added mid-season,
   etc.), add to the right dict.
6. Add winner / strike overrides to `WINNER_OVERRIDES`, `WINNER_BY_MARKET`,
   or `TOP_OUTCOME_NAMES` as needed.

## Workflow

- Default branch: `main`. Push to `main` → GitHub Pages deploys.
- Work on feature branches and open PRs. Typical branch naming:
  `cleanup/<topic>`, `fix/<topic>`, `feat/<topic>`.
- Before pushing: `npm run build` locally to catch Observable errors.
- The data CSVs change out-of-band when the owner runs the R pipeline. Don't
  commit stale CSVs.

## Non-goals / out of scope

- No server-side code.
- No mutation of Kalshi data — we only read and display the CSVs.
- No hosting of the private R pipeline in this repo.
- No "Ticker" column in the leaderboard (explicitly declined). Instead, raw
  ticker stays in the underlying data and the visible columns are
  human-readable.
